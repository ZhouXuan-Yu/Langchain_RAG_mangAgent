# 线程池架构设计方案

> 文档版本：v1.0
> 日期：2026-04-02
> 目标：解决多页面并发执行任务时的阻塞问题，优化整体架构

---

## 一、问题分析

### 1.1 当前问题

用户反映：**页面中执行了一个任务，其他页面的功能无法使用**。

通过代码分析，发现以下根本原因：

| 问题 | 位置 | 说明 |
|------|------|------|
| **单一事件循环阻塞** | `api.py` 的 `chat_stream` | `agent.astream_events()` 虽然是 async，但 LLM 推理本身是 CPU/IO 密集型，多个并发调用会互相争抢事件循环 |
| **后台任务缺乏隔离** | `orch_jobs.py` | `BackgroundJob` 是进程内单例，所有任务共享同一异步执行上下文 |
| **未充分利用线程池** | `task_scheduler.py` | `_executor = ThreadPoolExecutor(max_workers=8)` 已定义但**从未使用**，属于死代码 |
| **LLM 调用未线程隔离** | LLM 客户端 | DeepSeek/Claude 客户端的网络请求和 JSON 解析在主线程执行 |
| **SQLite 串行访问** | `sqlite_store.py` | 虽然 `check_same_thread=False`，但没有连接池，高并发下成为瓶颈 |
| **ChromaDB 同步操作** | `chroma_store.py` | 虽然用 `asyncio.to_thread()` 包装，但缺乏专用线程池管理 |

### 1.2 当前架构简图

```
┌──────────────────────────────────────────────────────────────────────┐
│                        FastAPI + Uvicorn                               │
│                         (单事件循环)                                    │
│                                                                       │
│   页面A请求 ──> chat_stream ──> astream_events ──┐                   │
│                                                    │ 共同竞争          │
│   页面B请求 ──> orchestrate ──> asyncio.gather ───┤ 同一个           │
│                                                    │ 事件循环         │
│   页面C请求 ──> memory_search ────────────────────┘                   │
│                                                                       │
│   问题：任何一个长时间 LLM 调用都会阻塞其他所有请求                    │
└──────────────────────────────────────────────────────────────────────┘
```

## 二、解决方案：线程池隔离架构

### 2.1 核心设计思想

引入**三层隔离架构**：

1. **API 层（Asyncio Event Loop）**：只负责接收请求、路由、返回响应，不执行任何耗时操作
2. **线程池层（Thread Pool）**：承担所有 CPU/IO 密集型任务（LLM 调用、向量检索、数据库操作）
3. **任务调度层（Task Scheduler）**：管理任务队列、优先级、依赖关系、生命周期

### 2.2 目标架构简图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        FastAPI (API Layer)                               │
│              只负责接收请求、分发任务、返回结果，不做耗时计算              │

│   POST /api/orchestrate ─────────┼──> TaskScheduler.submit()           │

│                                      │                                   │
│   GET /api/chat/stream/{job_id} ────┼──> 从 ResultStore 读取结果       │
```

#### 3.1.1 线程池管理器 (`src/utils/thread_pool_manager.py`)

**职责**：全局线程池生命周期管理

```python
class ThreadPoolManager:
    """全局线程池管理器"""
    
    llm_pool: ThreadPoolExecutor      # LLM 调用专用线程池
    db_pool: ThreadPoolExecutor         # 数据库操作专用线程池
    vector_pool: ThreadPoolExecutor     # 向量检索专用线程池
    
    def __init__(self):
        self.llm_pool = ThreadPoolExecutor(
            max_workers=4, 
            thread_name_prefix="llm_worker_",
            queue_size=50
        )
        self.db_pool = ThreadPoolExecutor(
            max_workers=8,
            thread_name_prefix="db_worker_"
        )
        self.vector_pool = ThreadPoolExecutor(
            max_workers=4,
            thread_name_prefix="vector_worker_"
        )
    
    def submit_llm(self, fn, *args, **kwargs) -> Future:
        """提交 LLM 调用到专用线程池"""
        return self.llm_pool.submit(fn, *args, **kwargs)
    
    def submit_db(self, fn, *args, **kwargs) -> Future:
        """提交数据库操作"""
        return self.db_pool.submit(fn, *args, **kwargs)
    
    def submit_vector(self, fn, *args, **kwargs) -> Future:
        """提交向量检索操作"""
        return self.vector_pool.submit(fn, *args, **kwargs)
    
    async def submit_llm_async(self, fn, *args, **kwargs):
        """异步方式提交 LLM 调用（兼容 async 接口）"""
        loop = asyncio.get_running_loop()
        return await loop.run_in_executor(self.llm_pool, lambda: fn(*args, **kwargs))
    
    def shutdown(self, wait=True):
        """优雅关闭所有线程池"""
        self.llm_pool.shutdown(wait=wait)
        self.db_pool.shutdown(wait=wait)
        self.vector_pool.shutdown(wait=wait)
```

#### 3.1.2 任务调度器 (`src/server/task_scheduler.py` 重构)

**职责**：统一的任务提交、队列管理、优先级调度

```python
from dataclasses import dataclass, field
from enum import Enum
from typing import Any, Callable, Optional
import asyncio
from concurrent.futures import Future
import uuid
import time

class TaskPriority(Enum):
    LOW = 3
    NORMAL = 2
    HIGH = 1
    URGENT = 0

class TaskStatus(Enum):
    PENDING = "pending"
    RUNNING = "running"
    DONE = "done"
    FAILED = "failed"
    CANCELLED = "cancelled"

@dataclass(order=True)
class Task:
    priority: int                      # 优先级（数字越小越优先）
    created_at: float = field(compare=True)
    job_id: str = field(compare=False, default="")
    task_type: str = field(compare=False, default="")  # "chat", "orchestrate", "memory"
    params: dict = field(compare=False, default_factory=dict)
    status: TaskStatus = field(compare=False, default=TaskStatus.PENDING)
    future: Optional[Future] = field(compare=False, default=None)
    result_store_key: str = field(compare=False, default="")


class TaskScheduler:
    """
    统一任务调度器
    
    核心设计：
    1. API 层只负责接收请求、生成 job_id、submit 到调度器、立即返回
    2. 所有耗时操作在独立线程池中执行
    3. 结果通过 ResultStore 存储，SSE 轮询读取
    """
    
    def __init__(self, pool_manager: ThreadPoolManager):
        self.pool_manager = pool_manager
        self._task_queue: asyncio.PriorityQueue = asyncio.PriorityQueue(maxsize=1000)
        self._running_tasks: dict[str, asyncio.Task] = {}
        self._result_store: dict[str, dict] = {}
        self._result_lock = asyncio.Lock()
        self._cancel_events: dict[str, asyncio.Event] = {}
        
        # 启动调度循环
        asyncio.create_task(self._schedule_loop())
        asyncio.create_task(self._health_check_loop())
    
    async def submit_chat_task(
        self, 
        message: str, 
        session_id: str,
        model: str,
        priority: TaskPriority = TaskPriority.NORMAL
    ) -> str:
        """提交聊天任务"""
        job_id = str(uuid.uuid4())
        
        await self._add_to_queue(
            job_id=job_id,
            task_type="chat",
            priority=priority,
            params={"message": message, "session_id": session_id, "model": model}
        )
        return job_id
    
    async def submit_orchestrate_task(
        self,
        requirement: str,
        enabled_agents: list[str],
        priority: TaskPriority = TaskPriority.NORMAL
    ) -> str:
        """提交编排任务"""
        job_id = str(uuid.uuid4())
        
        await self._add_to_queue(
            job_id=job_id,
            task_type="orchestrate",
            priority=priority,
            params={"requirement": requirement, "enabled_agents": enabled_agents}
        )
        return job_id
    
    async def _add_to_queue(
        self,
        job_id: str,
        task_type: str,
        priority: TaskPriority,
        params: dict
    ):
        """添加任务到队列"""
        task = Task(
            priority=priority.value,
            created_at=time.time(),
            job_id=job_id,
            task_type=task_type,
            params=params,
            status=TaskStatus.PENDING,
            result_store_key=job_id
        )
        
        # 初始化结果存储
        async with self._result_lock:
            self._result_store[job_id] = {
                "status": TaskStatus.PENDING,
                "progress": 0,
                "events": [],
                "result": None,
                "error": None,
                "created_at": time.time()
            }
        
        # 初始化取消事件
        self._cancel_events[job_id] = asyncio.Event()
        
        await self._task_queue.put(task)
    
    async def _schedule_loop(self):
        """
        调度循环 - 持续从队列取出任务并分发到线程池
        
        关键设计：
        1. 多个并发槽位（concurrent_slots=10），允许同时运行多个任务
        2. 按优先级调度，高优先级任务优先执行
        3. 每个任务在独立线程中执行，不阻塞调度循环
        """
        concurrent_slots = 10
        active_tasks: dict[str, asyncio.Task] = {}
        
        while True:
            # 清理已完成的协程
            done = [tid for tid, t in active_tasks.items() if t.done()]
            for tid in done:
                del active_tasks[tid]
            
            # 如果有空闲槽位，从队列取任务
            while len(active_tasks) < concurrent_slots:
                try:
                    task = await asyncio.wait_for(
                        self._task_queue.get(), 
                        timeout=0.5
                    )
                    
                    # 创建取消事件（如果没有）
                    if task.job_id not in self._cancel_events:
                        self._cancel_events[task.job_id] = asyncio.Event()
                    
                    # 创建执行任务
                    exec_task = asyncio.create_task(
                        self._execute_task(task)
                    )
                    active_tasks[task.job_id] = exec_task
                    
                except asyncio.TimeoutError:
                    break
    
    async def _execute_task(self, task: Task):
        """在独立线程中执行任务"""
        loop = asyncio.get_running_loop()
        
        async with self._result_lock:
            self._result_store[task.job_id]["status"] = TaskStatus.RUNNING
        
        try:
            if task.task_type == "chat":
                result = await self._run_chat_task(task)
            elif task.task_type == "orchestrate":
                result = await self._run_orchestrate_task(task)
            else:
                raise ValueError(f"Unknown task type: {task.task_type}")
            
            async with self._result_lock:
                self._result_store[task.job_id]["status"] = TaskStatus.DONE
                self._result_store[task.job_id]["result"] = result
                self._result_store[task.job_id]["progress"] = 100
                
        except asyncio.CancelledError:
            async with self._result_lock:
                self._result_store[task.job_id]["status"] = TaskStatus.CANCELLED
            raise
            
        except Exception as e:
            async with self._result_lock:
                self._result_store[task.job_id]["status"] = TaskStatus.FAILED
                self._result_store[task.job_id]["error"] = str(e)
        finally:
            self._cancel_events.pop(task.job_id, None)
    
    async def _run_chat_task(self, task: Task) -> dict:
        """在线程池中执行聊天任务（使用 run_in_executor）"""
        def _sync_chat():
            from src.server.dependencies import get_agent
            
            agent = get_agent(task.params["model"])
            config = {"configurable": {"thread_id": task.params["session_id"]}}
            
            result_chunks = []
            for event in agent.astream_events(
                {"messages": [HumanMessage(content=task.params["message"])]},
                config=config,
                version="v2"
            ):
                # 检查取消
                if self._cancel_events[task.job_id].is_set():
                    raise asyncio.CancelledError()
                
                result_chunks.append(event)
            
            return result_chunks
        
        # 在 LLM 线程池中执行
        return await asyncio.get_running_loop().run_in_executor(
            self.pool_manager.llm_pool,
            _sync_chat
        )
    
    async def _run_orchestrate_task(self, task: Task) -> dict:
        """在线程池中执行编排任务"""
        def _sync_orchestrate():
            from src.graph.orchestrator import CrayfishOrchestrator
            
            orch = CrayfishOrchestrator(...)
            return orch.orchestrate(...)
        
        return await asyncio.get_running_loop().run_in_executor(
            self.pool_manager.llm_pool,
            _sync_orchestrate
        )
    
    async def get_task_status(self, job_id: str) -> dict:
        """获取任务状态"""
        async with self._result_lock:
            return self._result_store.get(job_id, {"status": "not_found"})
    
    async def cancel_task(self, job_id: str):
        """取消任务"""
        if job_id in self._cancel_events:
            self._cancel_events[job_id].set()
    
    async def _health_check_loop(self):
        """健康检查循环 - 定期清理过期结果"""
        while True:
            await asyncio.sleep(300)  # 每5分钟
            async with self._result_lock:
                now = time.time()
                expired = [
                    jid for jid, data in self._result_store.items()
                    if now - data["created_at"] > 3600 and data["status"] in (TaskStatus.DONE, TaskStatus.FAILED)
                ]
                for jid in expired:
                    del self._result_store[jid]
```

```python
@router.get("/api/chat/stream/{job_id}")
async def chat_stream_sse(job_id: str):
    """
    SSE 流式获取聊天结果
    内部轮询 ResultStore，有新数据立即推送
    """
    async def generate():
        last_idx = 0
        while True:
            status_data = await task_scheduler.get_task_status(job_id)
            
            if status_data["status"] == "done":
                # 流式推送结果
                for chunk in status_data.get("result", []):
                    yield f"data: {json.dumps({'type': 'chunk', 'data': chunk})}\n\n"
                yield f"data: {json.dumps({'type': 'done'})}\n\n"
                break
            
            elif status_data["status"] == "failed":
                yield f"data: {json.dumps({'type': 'error', 'error': status_data.get('error')})}\n\n"
                break
            
            # 增量推送新事件
            events = status_data.get("events", [])
            while last_idx < len(events):
                yield f"data: {json.dumps(events[last_idx])}\n\n"
                last_idx += 1
            
            await asyncio.sleep(0.1)  # 100ms 轮询间隔
    
    return StreamingResponse(generate(), media_type="text/event-stream")
```

### 3.3 线程池在 LLM 客户端中的集成

#### 3.3.1 DeepSeek 客户端改造 (`src/llm/deepseek_client.py`)

```python
import asyncio
from concurrent.futures import ThreadPoolExecutor
from typing import AsyncIterator, Optional
import threading

class ThreadSafeDeepSeekClient:
    """线程安全的 DeepSeek 客户端"""
    
    def __init__(self, pool: Optional[ThreadPoolExecutor] = None):
        self._pool = pool or ThreadPoolExecutor(max_workers=4)
        self._client_lock = threading.Lock()  # 每个线程独立的 client 实例
    
    def _get_thread_client(self):
        """获取当前线程的 client 实例"""
        thread_id = threading.current_thread().ident
        if not hasattr(self, '_clients'):
            self._clients = {}
        
        if thread_id not in self._clients:
            self._clients[thread_id] = DeepSeekClient()  # 线程本地实例
        return self._clients[thread_id]
    
    async def astream(self, messages: list, **kwargs) -> AsyncIterator[str]:
        """异步流式调用（在独立线程中执行）"""
        loop = asyncio.get_running_loop()
        
        def _sync_stream():
            client = self._get_thread_client()
            for chunk in client.stream(messages, **kwargs):
                yield chunk
        
        return await loop.run_in_executor(self._pool, _sync_stream)
```

### 3.4 SQLite 连接池改造

#### 3.4.1 连接池实现 (`src/memory/sqlite_pool.py`)

```python
import sqlite3
import threading
from queue import Queue, Empty
from contextlib import contextmanager

class SQLitePool:
    """
    SQLite 连接池（针对每个数据库文件的连接复用）
    
    虽然 SQLite 本身是进程级别的，但通过连接池可以：
    1. 复用连接对象，减少连接创建开销
    2. 通过队列控制并发访问数量
    3. 提供上下文管理器，自动归还连接
    """
    
    def __init__(self, db_path: str, pool_size: int = 5):
        self.db_path = db_path
        self._pool: Queue = Queue(maxsize=pool_size)
        self._lock = threading.Lock()
        
        # 预创建连接
        for _ in range(pool_size):
            conn = self._create_connection()
            self._pool.put(conn)
    
    def _create_connection(self) -> sqlite3.Connection:
        conn = sqlite3.connect(
            self.db_path,
            check_same_thread=False,  # 允许多线程共享
            timeout=30.0
        )
        conn.row_factory = sqlite3.Row
        # 启用 WAL 模式以提高并发性能
        conn.execute("PRAGMA journal_mode=WAL")
        conn.execute("PRAGMA synchronous=NORMAL")
        conn.execute("PRAGMA busy_timeout=30000")
        return conn
    
    @contextmanager
    def get_connection(self):
        """获取连接（自动归还）"""
        conn = self._pool.get(timeout=30)
        try:
            yield conn
        finally:
            self._pool.put(conn)
    
    def close_all(self):
        """关闭所有连接"""
        while not self._pool.empty():
            try:
                conn = self._pool.get_nowait()
                conn.close()
            except Empty:
                break
```

### 3.5 任务取消机制

```python
class TaskCancellationToken:
    """任务取消令牌 - 支持树形取消（取消父任务同时取消子任务）"""
    
    def __init__(self, job_id: str):
        self.job_id = job_id
        self._cancelled = threading.Event()  # 线程安全取消标志
        self._children: dict[str, 'TaskCancellationToken'] = {}
        self._lock = threading.Lock()
    
    def cancel(self):
        """取消此任务及所有子任务"""
        self._cancelled.set()
        with self._lock:
            for child in self._children.values():
                child.cancel()
    
    def is_cancelled(self) -> bool:
        return self._cancelled.is_set()
    
    def check_and_raise(self):
        """检查取消状态，有则抛异常"""
        if self.is_cancelled():
            raise TaskCancelledError(self.job_id)
    
    def register_child(self, child_id: str, token: 'TaskCancellationToken'):
        """注册子任务"""
        with self._lock:
            self._children[child_id] = token
```

### 3.6 优先级队列调度

```python
class PriorityAwareScheduler:
    """支持优先级的任务调度器"""
    
    def __init__(self, max_concurrent: int = 10):
        self.max_concurrent = max_concurrent
        self._queues: dict[int, asyncio.Queue] = {
            p.value: asyncio.Queue() for p in TaskPriority
        }
        self._active_count = 0
    
    async def submit(self, priority: TaskPriority, coro):
        """提交任务（按优先级排队）"""
        await self._queues[priority.value].put(coro)
        await self._dispatch()
    
    async def _dispatch(self):
        """分发任务（优先处理高优先级队列）"""
        if self._active_count >= self.max_concurrent:
            return
        
        # 从高到低检查各优先级队列
        for pval in sorted(self._queues.keys()):
            queue = self._queues[pval]
            try:
                coro = queue.get_nowait()
                asyncio.create_task(self._run_task(coro))
                self._active_count += 1
                break
            except asyncio.QueueEmpty:
                continue
    
    async def _run_task(self, coro):
        """执行任务"""
        try:
            await coro
        finally:
            self._active_count -= 1
            await self._dispatch()  # 尝试调度下一个
```

## 八、注意事项

1. **Python GIL 限制**：Python 的 GIL 意味着 CPU 密集型任务无法通过线程池获得真正的并行。但 LLM 调用和网络 I/O 是 I/O 密集型操作，线程池能有效提升并发性能。

2. **线程数配置**：线程池大小需要根据 CPU 核心数和内存配置调整。过大的线程池反而会降低性能（线程切换开销）。

3. **LLM API 限流**：使用线程池时需要注意目标 API 的 QPS 限制。例如 DeepSeek API 可能有并发限制，需要在客户端层面控制。

4. **内存占用**：每个线程默认占用约 1MB 栈空间。4+8+4=16 个线程约 16MB 额外内存，是可接受的开销。

5. **向后兼容**：改造过程中需要保持 API 兼容，可以通过版本化 API（如 `/api/v2/chat/stream`）实现平滑迁移。
